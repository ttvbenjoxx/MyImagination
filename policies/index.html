MyImagination Privacy & Cookie Policy

Effective Date: 3/22/24

This Privacy & Cookie Policy outlines how MyImagination collects, uses, and protects user data. This site is designed with user privacy in mind and aims to provide full transparency regarding data practices.

1. Information Collected

MyImagination uses a third-party sign-in service to authenticate users. When a user signs in, the following basic profile information is collected:

  • Name
  • Email address

Passwords are not collected, stored, or processed by this site.

2. Use of Information

Collected information is used exclusively to:

  • Authenticate user identity
  • Display user details on forum content
  • Maintain user access and session stability
  • Ensure the integrity and security of user accounts

No data is used for advertising, analytics, or personalized marketing.

3. Cookies and Local Storage

MyImagination does not use cookies or any third-party trackers.

To support a seamless experience, this site uses local storage to retain user session data. This ensures that users remain signed in between visits. Only essential, non-sensitive data related to login status is stored locally.

4. Data Sharing and Third Parties

User data is never sold, rented, or shared with third parties. Information collected is used strictly for the purpose of enabling account access and forum functionality.

No external tools or integrations are used for analytics or advertising.

5. User Control and Data Access

Users may request access to or deletion of their account and associated personal data by contacting gmolter@ftstudent.org.

Note: Deleting an account does not remove posts, comments, or replies made on the forum. To request removal of specific content, users must complete the Post Deletion Request found on Page 2 of the MyImagination Issue Report Form, available here. All requests are reviewed and processed within a reasonable timeframe.

6. Data Security

Appropriate security measures, including HTTPS encryption and secure session handling, are in place to protect user information. No password data is collected or stored, minimizing the risk of credential-based threats.

7. Policy Updates

This policy may be updated from time to time to reflect platform changes or legal requirements. All updates will be posted on this page. Continued use of the site after any policy update constitutes acceptance of the revised terms.
